# AI4JOB 产品需求文档

# 必须了解总目标
这个文档是基于prdpro.md进行产品修改的迭代文档，要求每次修改不得偏离prdpro.md。

# revise1
对index.html:
1. 现在是一整个页面下滑可以看见所有功能页，但我想实现的是点击对应的才跳转到功能页中。
2. 无需特定的投递管理页，投递管理直接在首页以表格形式进行实现。
2. 对于首页，首页整体大小固定为屏幕大小，我的想法是它最上方左侧有3个小的导航页面栏：首页；简历优化；面试优化。最上方右侧是用户名（需要用户登录存储对应用户信息）；导航栏下方左侧是岗位上传入口，用户可以粘贴文本，或者粘贴链接，从而获得对应岗位的描述；导航栏下方右侧是所有用户增加过的所有岗位投递管理表格。表格上面可以进行一些筛选分类，表格包含字段如下，表格双击可以修改，表格内部按照上下滑动可以看见所有上传过的岗位。
| 字段 | 类型 | 来源 | 说明 |
| --- | --- | --- | --- |
|选择框|勾选选择这一条
|进入优化| 点击弹出三个选项：简历优化；面试优化；投递管理|
| 岗位 JD | 文本/链接 | 用户输入 | |
| 公司 | 文本 | 用户填写或 AI 补全 | 若用户已填写，AI 不覆盖 |
| 职位 | 文本 | 用户填写或 AI 补全 | 同上 |
| 地点 | 文本 | 用户填写或 AI 补全 | 同上 |
| 来源渠道 | 枚举/文本 | 用户填写 | 如官网、Boss、内推 |
| 投递链接 | URL | 用户填写 | 可为空 |
| 重视度 | 枚举 | 用户填写 | 高/中/低 |
| 标签 | 多选 | 用户填写 | 如急招、内推、已沟通 |
| 进展状态 | 枚举 | 用户维护 | 见状态机定义 |
| 投递日期 | 日期 | 用户填写 | 可为空 |
| 下一节点 DDL | 日期时间 | 用户填写 | 用于记录下一关键节点截止时间或预约时间 |
| 最后更新时间 | 日期时间 | 系统生成 | 任意状态或内容变更时更新 |
| 备注 | 长文本 | 用户填写 | 记录补充信息 |
|删除|点击弹出确定删除吗？确定即删除这一条及其岗位空间内容
3. 对于简历优化和面试优化，有两个入口。以简历优化为例：①首页上方导航栏的“简历优化”；②首页投递管理表格的每一行的“进入优化”按钮。
4. 简历优化界面与面试优化界面具体内容暂停止修改，保留接口，先实现首页。
5. 你可以自行接入更适合的代码形式或内容，保持上述需求要求，技术细节由你自己决定。

# revise2
1. 现在页面是只有中间显示，并没有占满整个页面，左右有留白，修改。
2. 岗位上传入口应该是文本链接2选1，不是现在两个都同时出现，也就是一个框，用户粘贴文本，就自动走文本识别，用户粘贴链接，就自动去对应链接识别。公司、职位、地点、来源通道不需要在岗位上传入口处显示，只在投递管理处显示，由ai自动补充，如果不对用户可以双击修改。
3. 左侧下方的快速筛选移到右侧投递管理上方。
4. 所有中文字体采用微软雅黑，英文字体采用新罗马。
5. 删除投递管理中每一行的进入优化按钮。
6. 将每一行的删除按钮移除，放在投递管理表格右上角，当勾选之后（一条或多条），点击删除后把对应行删掉。
7. 每一行的岗位jd放在投递链接后，并且在首页只显示“详情”，点击详情进行详细展示。
8. 投递管理每一行都只显示一行，不要换行显示。
9. 删除首页右上角的“当前任务****”。
10. 左侧固定大小，无需上下滑动。
11. 投递管理表格每个表头都增加一个排序的，点击就自上而下排序，再点击就自下而上排序。
