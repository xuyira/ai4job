# revise5
简历优化页面的基于大模型的简历优化 workflow 设计：
1. 岗位分析
输入JD、岗位相关资料（面经、行业动态、其他），输出结构化岗位分析，回答4个关键问题：
- 结构化岗位分析：核心职责、必备技能、加分项、关键词、经验年限要求、结构导向指标
- 关键问题：
①这个岗位的核心工作是什么？日常主要做哪些事？
②这个岗位的核心特点是什么？和其他相似岗位有什么区别？
③该业务线的目标方向是什么？该岗位在其中的核心价值是什么？
④为了胜任这个岗位，我需要具备哪些核心能力？（或什么样的候选人最适配？）
2. 简历分析
输入简历、个人相关资料，输出基于岗位画像的简历结构化评分：
- 结构化简历评分：
①岗位匹配（40%）：匹配度（JD对齐）、关键词覆盖（ATS友好）、经验相关性；
②内容质量（30%）：清晰度、结构规范性、内容完整性、专业性；
③说服力（30%）：成果量化、项目表述力度、竞争力（差异化）。
- 扣分原因
- 从技术和业务两个角度来看，我能申请这个岗位吗？需要重点做哪些适配？
3. 逐条生成简历优化建议（这里应当还能输入一条用户的目标：想强调什么经历、不想改什么内容等）
每条建议都必须是结构化的：
- suggestion_id
- 针对哪一段
- 修改前内容
- 修改原因（基于JD、岗位相关资料、简历、个人相关资料）
- 修改后建议版本
- 预期收益
- 风险提示
- 优先级
4. 用户交互确认
前端支持每条建议：
- 接受
- 拒绝
- 自己手动编辑
- 用户给出修改意见，让模型依据意见重写这一条（或直接重写）
- 查看为什么这么改
5. 生成新简历
只对已采纳部分生效，合成新版本
6. 新简历复评分
再次按照同一套rubric打分，输出：
- 提升了哪些维度
- 哪些问题还存在
